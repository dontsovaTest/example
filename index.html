Фиксим баг с GIT доступом. В случае если данным компом пользовались до вас и пользовались именно гитом, может появится проблема с подключением своего ГИТА. Для начала надо зайти в папку с расширение .ssh (Это скрытая папка, поэтому чтобы ее увидеть вам понадобится в настройка сделать скрытые папки видимыми) Для MacBooc это комбинация клавиш Cmnd+Shift+. И в этой папке удаляем все ssh ключи PS для винды путь такой: C:/User/Class5-Pc10/.ssh Я уже раньше добавляла ключ на этот комп я его удалила, поэтому изначально гит мне подсказывает что у меня был ключ Но затем говорит что у меня нету прав Теперь мы должны подвязать ssh ключ Прежде чем ключ создасться гит спросит у вас куда его сохранить, ничего не вводим пусть сохраняется в папке по дефолту Потом спрашивает будет ли какой-то пароль для этого ключа, можно ввести, я не вводила затем повторить пароль, опять так я не вводила теперь нам надо посмотреть какой ключ был сгенерирован чтобы его вставить в гитхаб Your identification has been saved in /Users/air/.ssh/id_ed25519 - этот не трогаем!!!! Your public key has been saved in /Users/air/.ssh/id_ed25519.pub - нужен именно с ресширением .pub, то есть вот этот чтобы внего попасть вбивает в терминале команду cat (она открывает файлы показывая что в них, по другому не увидите вметимое файла) гит уточнил что раньше не встречал этот ключ, вы уверены что хотите продолжить и я нажала да не забудьте перед этим настроить git config git config user.name "your name on github" git config user.email "your email on github" и иногда надо обязательно еще прописывать глобально обычно гит сам может подсказать когда это надо git config --global user.name "your name on github" git config --global user.email "your email on github" В целом это все, у меня все залилось на гит
